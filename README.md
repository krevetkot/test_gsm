# csvutility

Программа читает CSV-файл с формулами, вычисляет значения ячеек и выводит результат в консоль в CSV-формате.

## Формат входного файла

Первая строка — заголовок: пустая ячейка, затем имена столбцов через запятую.
Каждая следующая строка начинается с целого положительного номера строки (>0), затем идут значения ячеек.
Номера строк могут идти в произвольном порядке.

Ячейка может содержать:
- целое число (в том числе отрицательное)
- формулу вида `=ARG1 OP ARG2`, где `ARG1` и `ARG2` — целые числа или адреса ячеек, а `OP` — один из операторов `+`, `-`, `*`, `/`

Адрес ячейки записывается как `ИмяСтолбца НомерСтроки`, например `A1`, `Cell30`.

Пример файла:

```
,A,B,Cell
1,1,0,1
2,2,=A1+Cell30,0
30,0,=B1+A1,5
```

Результат:

```
,A,B,Cell
1,1,0,1
2,2,6,0
30,0,1,5
```

## Сборка

Требования: компилятор `gcc` или `clang`, утилита `make`.

```bash
make
```

Собранный бинарник — `csvutility` в текущей директории.

Для сборки вручную без make:

```bash
gcc -Wall -Wextra -std=c99 -pedantic -O2 -o csvutility main.c csvutility.c
```

## Запуск

```bash
./csvutility file.csv
```

## Тесты

```bash
make test
```

Тесты проверяют как корректный вывод, так и коды возврата при ошибках:

| Тест | Описание | Ожидаемый exit code |
|------|----------|-------------------|
| 1 | Базовый пример из задания | 0 |
| 2 | Целочисленная арифметика | 0 |
| 3 | Отрицательные числа | 0 |
| 4 | Деление на ноль | 3 |
| 5 | Циклическая зависимость | 3 |
| 6 | Несуществующий адрес ячейки | 3 |
| 7 | Строки не в порядке возрастания | 0 |
| 8 | Файл не существует | 1 |
| 9 | Неверный заголовок | 2 |
| 10 | Дублирующийся номер строки | 2 |
| 11 | Несовпадение числа столбцов | 2 |

Вывод тестов, ожидающих корректное завершение программы, сравнивается с эталонным выводом в папке /test/out.

## Коды возврата

| Код | Константа | Описание |
|-----|-----------|----------|
| 0 | `CSV_OK` | Успешное выполнение |
| 1 | `CSV_ERR_IO` | Ошибка ввода-вывода (файл не найден, пустой файл) |
| 2 | `CSV_ERR_FORMAT` | Ошибка формата (плохой заголовок, дубликаты, несовпадение столбцов) |
| 3 | `CSV_ERR_EVAL` | Ошибка вычисления (деление на ноль, цикл, неверная ссылка) |

Сообщения об ошибках выводятся в `stderr`.

## Детали реализации

**Поиск столбцов и строк** выполняется за O(1) через две хэш-таблицы с открытой адресацией: `ColMap` (ключ — имя столбца, хэш djb2) и `RowMap` (ключ — номер строки, целочисленный хэш). Ёмкость таблиц — 512 слотов при максимуме 256 элементов.

**Вычисление формул** рекурсивное с обнаружением циклов: ячейка помечается как «в процессе» (`computed = -1`) до завершения вычисления. Если при рекурсивном обходе встречается такая ячейка — фиксируется циклическая зависимость.

**Обработка ошибок** построена на макросах `CSV_FAIL_IO`, `CSV_FAIL_FORMAT`, `CSV_FAIL_EVAL`. Каждый макрос одновременно печатает сообщение в `stderr` и возвращает соответствующий код. Ошибки пробрасываются вверх по стеку вызовов через возвращаемое значение типа `CsvError`.
